# Issue #002: Dev server 占用生产端口

**日期**: 2026-02-07
**严重性**: 高
**状态**: 已修复

## 问题

初始化 PixiJS 项目时，将 Vite dev server 配置为 port 3000 并直接启动，占用了用户 CC Dashboard 正在使用的端口。`pkill -f "vite"` 也可能误杀其他进程。

## 根本原因

1. 启动服务前未检查端口占用
2. 不应该在验证阶段启动 dev server，typecheck 就够了
3. 违反了"不要直接影响生产环境"的规则

## 修复

- Vite 端口改为 5173（默认值），`strictPort: false` 自动回退
- 移除 `open: true` 避免自动打开浏览器

## 教训

1. **永远不要假设端口空闲** — 启动任何服务前必须 `lsof -i:<port>` 检查
2. **验证项目不需要起服务** — typecheck + build 足够证明项目可用
3. **不要在用户机器上随意启动/杀死进程** — 这等于操作生产环境
